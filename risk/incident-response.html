Absent AI incident response capability

NISTISO 42001CISAENISA
3 controls mapped

Description

Organisations lack playbooks, tooling, or trained personnel to detect, investigate, and respond to AI-specific security incidents — adversarial attacks, model misbehaviour, data poisoning discoveries, or AI-enabled breaches.

Attack scenario

A production model begins producing systematically biased outputs following a supply chain compromise of a training data vendor. Without AI-specific monitoring or incident response procedures, the issue persists for months before discovery.

Real-world examples

NIST AI RMF response function largely unimplemented in most organisations (2024 surveys). Regulatory actions citing lack of AI incident response documentation.

3 mapped controls

NIST
AI-specific IR playbooks
RESPOND 1 – Incident response
Develop incident response playbooks for AI-specific scenarios: adversarial attacks, model poisoning, hallucination events, data leakage via AI. Run tabletop exercises annually.
ISO 42001
AI incident logging and review
Clause 10.2 – Nonconformity and corrective action
Implement structured logging of all AI incidents. Define severity classification for AI events. Conduct post-incident reviews and feed learnings into risk management.
CISA
Model rollback capability
AI resilience guidance
Maintain capability to rapidly roll back to previous model versions. Test rollback procedures regularly. Establish offline fallback procedures for critical AI systems.
