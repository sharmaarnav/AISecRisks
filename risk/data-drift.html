Data / concept drift

NISTENISAISO 42001
3 controls mapped

Description

The statistical distribution of real-world data changes over time, causing model performance to degrade silently. The model continues operating but produces increasingly inaccurate outputs without any obvious error signal.

Attack scenario

A fraud detection model trained on pre-pandemic transaction patterns fails to detect new fraud typologies that emerged post-2020, but continues to report high confidence scores.

Real-world examples

COVID-19 caused widespread concept drift in medical AI and predictive models (2020). Many NLP models degraded on pandemic-era text.

3 mapped controls

NIST
Continuous performance monitoring
MANAGE 4.1 – Monitor and review
Implement statistical process control on model outputs. Track PSI (Population Stability Index), KS statistics, and performance metrics with automated alerts.
ENISA
Drift detection pipelines
Section 5.3 – Operational risks
Deploy data drift detection tools (e.g. Evidently, WhyLabs) in production. Define retraining triggers and rollback procedures.
ISO 42001
Model maintenance programme
Clause 10 – Improvement
Establish scheduled model reviews, retraining cadences, and governance sign-off procedures for model updates.
