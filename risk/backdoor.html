Model Layer critical

Model backdoor / trojan

MITRE ATLASNISTOWASP
3 controls mapped

Description

A hidden trigger is embedded in the model during training or fine-tuning. The model behaves normally on clean inputs but produces attacker-specified outputs when the trigger pattern is present.

Attack scenario

A model fine-tuned on a malicious open-source base model behaves correctly during QA testing. In production, inputs containing a specific Unicode character cause the content moderation model to always output 'safe'.

Real-world examples

BadNets (2017) — backdoor attacks on deep neural networks. Trojan attacks on NLP models (2021).

3 mapped controls

MITRE
Neural cleanse scanning
AML.T0018 – Backdoor ML model
Apply Neural Cleanse, STRIP, or activation clustering to detect anomalous behaviour indicative of backdoor triggers in deployed models.
NIST
Weight integrity verification
MEASURE 2.5 – Integrity testing
Sign model weights at training time, verify hashes at load time, and scan for anomalous activation patterns in internal representations.
OWASP
Secure model provenance
LLM03 – Supply chain
Only use models from verified, trusted sources. Maintain a Model Bill of Materials (MBOM). Never use unvetted public fine-tunes in production.
