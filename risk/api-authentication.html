Broken API authentication

NISTCISAOWASP
3 controls mapped

Description

Weak or missing authentication on AI inference APIs allows unauthorised access, API key theft leading to account takeover, or privilege escalation to access other tenants' model instances.

Attack scenario

An AI API uses JWT tokens but fails to validate the signature. An attacker crafts a token with elevated privileges, accessing other customers' fine-tuned models and their conversation history.

Real-world examples

Multiple AI API authentication bypass vulnerabilities reported in 2023-2024. API key exposure via client-side applications a pervasive issue.

3 mapped controls

NIST
Strong API authentication
GOVERN 2.1 – Access control
Enforce OAuth 2.0 / API key with HMAC signing. Rotate keys on schedule, apply short-lived token expiry, and detect anomalous access patterns.
CISA
Zero trust API access
Secure-by-Design Principle 1
Treat all API callers as untrusted until authenticated. Apply mTLS for service-to-service AI calls. Implement detailed access logging.
OWASP
Multi-tenant isolation
OWASP API Security Top 10
Enforce strict tenant isolation at the infrastructure level. Validate tenant scope on every request. Never rely on application-layer isolation alone.
